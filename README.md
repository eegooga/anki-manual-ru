## Рекомендации по стилю

В 2024 году мы начали работу по стандартизации стиля, используемого во всей документации. Вы, вероятно, заметите, что эти соглашения применяются в руководстве не везде последовательно, но это правила, которых мы придерживаемся в дальнейшем. Если вы отправляете правки или обновления, пожалуйста, следуйте этим рекомендациям. Также вы можете обновлять и другой текст, приводя его в соответствие с этими правилами.

- Используйте двойные кавычки `"` вместо одинарных кавычек `'`.
- Для выделения, например при определении термина, используйте курсив `_`.
- Также используйте курсив, когда речь идёт о самом слове, а не о его значении, например: «Чтобы искать либо _dog_, либо _cat_, используйте `dog or cat`».
- При упоминании элементов интерфейса используйте полужирное форматирование `**`.
- При упоминании клавиш клавиатуры оборачивайте их в `<kbd></kbd>`.
- Для примеров ввода используйте форматирование кода, оборачивая текст в обратные кавычки `` ` ``.

## Сохранение устаревших ссылок

### Удаление раздела

Мы часто используем якорные ссылки, чтобы направить пользователей к наиболее полезному для них разделу страницы. Эти якорные ссылки создаются из заголовков разделов, например: `https://docs.ankiweb.net/deck-options.html#timers`.

Если вы отправляете PR, который изменяет или удаляет заголовок раздела, пожалуйста, добавьте скрытый якорь, чтобы старые якорные ссылки продолжали работать корректно.

Пример: чтобы изменить заголовок «Clocks» на «Timers», замените:
```
## Clocks
```
на:
```
## Timers
<a id="Clocks"></a>
```

### Удаление страницы

Если вы отправляете PR, в результате которого страница полностью удаляется из руководства или FAQ:

1. Удалите соответствующую «главу» из [SUMMARY](src/SUMMARY.md) в руководстве [или FAQ: [SUMMARY](../../../faqs/blob/main/src/SUMMARY.md)].

2. Подумайте, нужно ли перенаправить ссылку на новое место в документации, или она должна завершаться [ошибкой 404](https://docs.ankiweb.net/404.html). Если нужно перенаправление, добавьте запись в секцию `[output.html.redirect]` файла [book.toml](book.toml) в руководстве [или FAQ: [book.toml](../../../faqs/blob/main/book.toml)].
   * Пример: `"old-page-name.html" = "new-page-name.html"`
   * См. [документацию mdBook](https://rust-lang.github.io/mdBook/format/configuration/renderers.html#outputhtmlredirect).
